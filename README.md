# Monorepo vs Monolith vs Multirepo

> ## 개요
>
> > - 개인이 작업하는 작은 프로젝트나 레포지토리 하나로 관리가 가능하다면 큰 문제는 없다. 하지만 대기업과 같은 큰 회사에서 진행하는 큰 프로젝트라면 수많은 각각의 기능들을 분리하고 몇개이상의 레포지토리에서 관리함으로써 코드가 어떤 레포지토리에 있는 코드를 참조하고 있는지 알기 쉽지 않고, 개발과 프로젝트 관리에 큰 어려움이 있을 수 있다. 이런 문제점을 보완하고자 Monorepo, Monolith, Multirepo라는 기법들이 등장했다.
